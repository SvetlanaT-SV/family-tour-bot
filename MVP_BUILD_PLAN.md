## ПРИНЦИПЫ РАЗРАБОТКИ

1. **Каждый шаг = видимый результат** — после теста что-то работает и это можно потрогать
2. **Сначала просто, потом красиво** — сначала работает, потом улучшаем
3. **Комментарии в коде** — каждая строка объяснена
4. **Проверка после каждого шага** — есть тест `test_stepN.py`
5. **Безопасность** — все токены только в `.env`, никогда в коде
