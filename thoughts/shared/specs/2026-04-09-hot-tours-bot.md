## Executive Summary

Автоматизированная система для турагентства, которая:
1. Парсит горящие туры с Турвизора по заданным фильтрам
2. Публикует красивые посты 2-3 раза в день в ВК, MAX, Telegram
3. Собирает заявки от клиентов через бота в мини-CRM
4. Хранит базу клиентов для будущего догрева и рассылок

**Для кого:** Руководитель турагентства, 3 менеджера  
**Почему сейчас:** Есть 5700 подписчиков в ВК и 500-2000 клиентов в базе — тёплая аудитория не активирована. Каждый непубликуемый горящий тур = упущенная продажа.

---

## Проблема

- Руководитель не успевает вручную искать и публиковать горящие туры
- Горящие туры сейчас **не публикуются вообще** — ноль продаж с соцсетей
- Холодная реклама в ВК: 10 000₽ → 20 лидов → 0 продаж
- Тёплая аудитория (5700 чел. в ВК) не получает актуальных предложений
- Данные о лидах нигде системно не хранятся для повторных продаж

---

## Критерии успеха

| Метрика | Цель (месяц 1) |
|---------|---------------|
| Продажи с бота | 1-2 тура |
| Заявки в боте | 10-20 |
| Посты в день | 2-3 |
| Время на публикации | 0 минут (автопилот) |
| Клиентов в базе бота | 20+ |

## Пользователи системы

### 1. Руководитель (основной)
- Получает уведомления о новых заявках
- Одобряет посты (первая неделя), потом автопилот
- Управляет фильтрами туров
- Просматривает базу клиентов

### 2. Менеджеры
- Получают назначенные заявки
- Обновляют статус клиента после звонка

### 3. Клиент
- Видит пост с горящим туром в ВК/MAX/Telegram
- Нажимает кнопку → попадает в бота
- Отвечает на 4-5 вопросов
- Получает подтверждение и ждёт звонка

## Пользовательский путь

### Путь клиента:
```
Увидел пост → Нажал "Узнать подробнее" → Открылся бот
→ Бот: "Как вас зовут?"
→ Бот: "Ваш телефон для связи?"
→ Бот: "Какие даты вам подходят? (или гибкие)"
→ Бот: "Сколько человек едет? Есть дети?"
→ Бот: "Примерный бюджет на человека?"
→ Бот: "Отлично! Менеджер свяжется с вами в течение 1 часа"
→ Руководителю → уведомление с данными
→ Менеджер звонит вручную
```

### Путь руководителя (режим проверки, неделя 1):
```
Бот нашёл горящий тур → Сформировал пост
→ Прислал на одобрение руководителю
→ Руководитель: ✅ или ❌
→ При ✅ → публикация во все каналы
```

### Путь руководителя (автопилот, с недели 2):
```
Бот нашёл тур → Сформировал пост → Опубликовал автоматически
→ Руководителю уведомление: "Опубликован тур: Турция, 45 000₽, 15 апреля"
```

## Функциональные требования

### P0 — Обязательно (запуск):

#### Парсер туров
- [ ] Получение туров через Tourvisor API (или парсинг)
- [ ] Фильтрация по параметрам:
  - Цена ниже заданного порога (настраивается)
  - Скидка от обычной цены (настраивается, напр. -25%)
  - Направления (список стран/городов)
  - Даты вылета (ближайшие N дней)
  - Города вылета (несколько)
- [ ] Получение данных отеля: название, звёзды, питание, расстояние до моря
- [ ] Получение фото отеля

#### Генератор поста
- [ ] Шаблон поста с:
  - Заголовок с эмодзи и страной
  - Цена (жирно, заметно)
  - Даты вылета и количество ночей
  - Город вылета
  - Название отеля, звёзды, питание
  - Краткое описание (из данных Турвизора)
  - 3-5 ключевых преимуществ (ИИ генерирует по данным отеля)
  - Призыв к действию (кнопка или ссылка)
- [ ] Генерация описания и преимуществ через Claude API

#### Публикатор
- [ ] Публикация в группу ВК через VK API
- [ ] Публикация в MAX-канал (через MAX API или вручную)
- [ ] Публикация в Telegram (через Bot API)
- [ ] Расписание: 09:30, 14:00, 19:00 (настраивается)
- [ ] Режим одобрения: пост на проверку руководителю перед публикацией
- [ ] Режим автопилота: публикует сам, присылает уведомление

#### Бот сбора заявок (Telegram + ВК)
- [ ] Диалог сбора данных (5 шагов):
  1. Имя
  2. Телефон
  3. Какой тур интересует (автоматически из поста)
  4. Даты / гибкость
  5. Состав туристов + бюджет
- [ ] Подтверждение клиенту: "Ждите звонка в течение часа"
- [ ] Уведомление руководителю: все данные клиента + ссылка на пост

#### База клиентов (мини-CRM бота)
- [ ] Хранение по каждому клиенту:
  - Имя, телефон, мессенджер/источник
  - История запросов (какие туры смотрел, когда)
  - Бюджет, состав туристов
  - Статус: новый / думает / купил / отказался
  - Дата последнего контакта
- [ ] Запись в Google Sheets (интеграция с "Мои документы туризм" — если есть API)
- [ ] Уведомление руководителю в Telegram при каждой заявке

### P1 — Важно (неделя 2-3):

- [ ] Панель управления для руководителя (Telegram-бот):
  - Просмотр новых заявок
  - Смена статуса клиента
  - Просмотр опубликованных туров
  - Включение/выключение автопилота
- [ ] Фильтры поиска туров (настройка через бота без кода)
- [ ] Сегментация базы по направлениям и бюджету (для будущих рассылок)
- [ ] Антидубль: не публиковать один тур дважды

### P2 — Желательно (месяц 2):

- [ ] Рассылка по базе (по сегментам)
- [ ] Автоответы на комментарии под постами в ВК
- [ ] Статистика: сколько постов, заявок, продаж
- [ ] A/B тест форматов постов

## Техническая архитектура

### Стек технологий (минимальный, бесплатный старт):

```
Язык:         Python 3.11
Планировщик:  APScheduler (запуск по расписанию)
Хостинг:      Railway.app (бесплатный тариф) или Render.com
База данных:  Google Sheets (через gspread) — бесплатно
Telegram бот: python-telegram-bot
ВК API:       vk_api (публикация постов)
ИИ тексты:   Claude API (claude-haiku — дёшево)
Турвизор:     Tourvisor API (у них есть API для агентств)
```

### Схема данных клиента:
```
Client {
  id:              уникальный ID
  name:            Имя
  phone:           Телефон
  messenger:       telegram_id / vk_id
  source:          откуда пришёл (пост ID)
  status:          new | thinking | bought | refused
  budget:          примерный бюджет
  tourists:        состав (взрослые/дети)
  inquiries: [     история запросов
    {
      tour_id:     ID тура
      country:     страна
      dates:       даты
      price:       цена
      date:        дата запроса
    }
  ]
  last_contact:    дата последнего контакта
  created_at:      дата регистрации
}
```

### Схема тура для публикации:
```
Tour {
  tourvisor_id:   ID в Турвизоре
  country:        Страна
  city_from:      Город вылета
  hotel_name:     Отель
  stars:          Звёзды
  meal:           Питание (AI, BB, HB, FB, UAI)
  nights:         Количество ночей
  date_from:      Дата вылета
  price:          Цена (за двоих)
  original_price: Обычная цена
  discount_pct:   Скидка %
  distance_sea:   До моря (м)
  photo_url:      Фото
  description:    Описание (из Турвизора)
  advantages:     [преимущества] (генерирует ИИ)
  published_at:   Когда опубликован
  published_ids:  {vk: id, tg: id, max: id}
}
```

## Нефункциональные требования

- **Надёжность:** Бот работает 24/7, даже если телефон выключен
- **Скорость:** Уведомление руководителю — в течение 30 секунд после заявки
- **Простота:** Всё управление через Telegram-бота, без кода
- **Безопасность:** Токены API хранятся в переменных окружения, не в коде
