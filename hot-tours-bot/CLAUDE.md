# Family Tour Bot — карта проекта

Турагентство Pegas Touristik (Уфа, опыт 12+ лет). Бот публикует горящие туры и новости туристической индустрии в три канала, собирает заявки клиентов.

## Стек
- Python 3.11, python-telegram-bot 21.9
- **Хостинг: Beget VPS** (`155.212.190.164`, Ubuntu 24.04, 480₽/мес, оплата картой Мир/СБП).
  Деплой ручной через SSH: `git pull && docker build && docker run`. Авто-CD пока не настроен.
  GitHub репозиторий — `SvetlanaT-SV/family-tour-bot`.
- AI: GigaChat (основной), Claude Anthropic (fallback), шаблон (последний fallback)
- Скрапинг описаний отелей: BeautifulSoup4 на tophotels.ru (поиск через DuckDuckGo HTML)
- Скрапинг новостей: BeautifulSoup4 на публичных Telegram-каналах (t.me/s/{name})
- Хранилище данных: Google Sheets (gspread) — листы "Туры к публикации", "Расписание",
  "Источники новостей", "Заявки", "Метаданные", "Описания отелей", "Журнал ошибок"
- Картинки: Pillow + DejaVu Sans Bold (для overlay на фото туров)

## Каналы публикации
- **Telegram-канал** family_tour_channel — `Config.TELEGRAM_CHANNEL_ID`, HTML-разметка работает
- **VK-группа** family_toor (id=60704869) —
  · `Config.VK_TOKEN` — групповой токен, для `wall.post` (пост от имени группы)
  · `Config.VK_USER_TOKEN` — пользовательский токен админа группы со scope `photos,wall,offline,groups`, для `photos.getWallUploadServer` / `photos.saveWallPhoto`
  Без user-токена VK возвращает ошибку 27 "method is unavailable with group auth", и фото падает на fallback (URL в attachment, без оверлея). С USER-токеном фото нормально грузится со всеми наложениями.
- **MAX-канал** — `Config.MAX_TOKEN`, `Config.MAX_CHAT_ID`, поддерживает только `<b>` `<i>`. Фото грузится через `/uploads?type=image` — работает с overlay-плашкой.

## Ключевые потоки

### 1. Публикация тура (основной поток)
1. Менеджер заполняет строку в Google Sheets `Туры к публикации` (статус "НОВЫЙ")
2. `publish_from_sheets()` в main.py каждые 5 минут проверяет лист
3. Для НОВЫХ туров: вызов `generate_post_from_dict()`. Источники фактов об отеле по приоритету:
   1) Ручная колонка «Особенности отеля» из Sheets
   2) Кэш в листе «Описания отелей» (ранее уже искали)
   3) Скрапинг tophotels.ru на лету + запись в кэш
   Затем GigaChat пишет текст по промпту, учитывая случайный стиль заголовка и
   случайные углы преимуществ. Постобработка вычищает воду и приводит к стандартной форме.
4. Скачивает фото → накладывает overlay → отправляет всем админам с тремя кнопками:
   - ✅ Сейчас → `_handle_approval` → `publish_to_channels` → TG/VK/MAX немедленно
   - ⏰ По расписанию → запись в Sheets лист "Расписание" → check_scheduled_posts публикует в ближайший слот 9/14/19 МСК
   - ❌ Пропустить
5. После одобрения статус в Sheets меняется на ОПУБЛИКОВАН

### 2. Сбор новостей (Stage 2)
1. Ежедневно в 08:00 МСК: `collect_news_job()` читает лист "Источники новостей" (только активные каналы)
2. Из каждого канала забирает посты за последние 24 часа через `fetch_channel_posts()`
3. Все посты передаются в `select_and_rewrite()` → GigaChat выбирает топ-3 и переписывает в наш стиль
4. Каждая новость отправляется админам как превью с теми же кнопками ✅/⏰/❌
5. В превью есть строка "🔗 Источник: @канал" — для проверки, в самом посте при публикации её НЕТ
6. Команда `/news` запускает сбор вручную не дожидаясь утра
7. **Догон пропусков:** после успешного сбора в "Метаданные" пишется `news_last_run_msk = YYYY-MM-DD`. При старте бота (post_init) проверяется: если время уже после 08:00 МСК и last_run ≠ сегодняшняя дата — через 60 секунд запускается one-shot джоба. Это защищает от пропусков когда Railway передеплоился после 08:00.

#### Что НЕ должно попадать в подборку (зашито в промпт processor.py)
- **B2B/операционка:** новые формы оферты, изменения договоров, кадровые перестановки, отчёты, презентации
- **Скидки авиакомпаний на билеты** (Аэрофлот, S7, Победа) — мы продаём пакетные туры, а не билеты
- Реклама агентств-конкурентов, политика, скандалы без угрозы туристу

### 3. Заявка клиента
- Клиент пишет одному из ботов → 5 вопросов (имя, телефон с валидацией, даты, состав, бюджет)
- Сохранение в Sheets лист "Заявки" + Telegram-уведомление всем `TELEGRAM_ADMIN_IDS`

### 4. Расписание публикаций
- Все запланированные посты идут в Google Sheets лист "Расписание" (а не локальный JSON)
- Это переживает перезапуски Railway (диск ephemeral) — раньше очередь терялась
- check_scheduled_posts каждую минуту читает Sheets, публикует те у которых наступило время, ставит "ОПУБЛИКОВАН"
- Слоты публикации: `Config.PUBLISH_HOURS = [9, 14, 19]` МСК

### 5. Восстановление после перезапуска
- `pending_posts.json` локально — пишется в файл рядом с кодом контейнера, переживает рестарты контейнера
- При полной пересборке (`docker rm + docker build + docker run`) файл теряется → `_handle_approval` восстанавливает текст и фото из самого Telegram-сообщения (`msg.text_html`, `msg.photo`)
- Защита от двойной публикации — проверка маркера "ОПУБЛИКОВАНО"/"ПРОПУЩЕН" в caption сообщения

## Колонки Google Sheets

### Лист "Туры к публикации"
Все ищутся по имени, можно переставлять:
`Статус | Страна | Курорт | Отель | Питание | Дата вылета | Ночей | Цена/чел | Город вылета | Особенности отеля | Фото URL | Ссылка | Опубликован | Ошибка`

- **Город вылета** — Уфа (по умолчанию), Казань, Москва и т.д. Бот склоняет в родительный.
- **Особенности отеля** — свободный текст реальных удобств ("первая линия, аквапарк, ультра все включено"). GigaChat использует ТОЛЬКО эти данные, не выдумывает.

### Лист "Расписание"
`Когда | Когда МСК | tour_id | Статус | Страна | Цена | Дата вылета | Текст | Photo URL | Photo bytes | Overlay страна | Overlay цена | Overlay вылет`
Статусы: ОЖИДАЕТ → ОПУБЛИКОВАН (или ОТМЕНЁН).

### Лист "Источники новостей"
`Канал | Категория | Активен`
- **Канал** — `https://t.me/atorus_news` или просто `atorus_news`
- **Активен** — `да` или `нет` (только активные обрабатываются)
- Только публичные каналы — Bot API не имеет доступа к закрытым

### Лист "Метаданные"
`Ключ | Значение` — служебное key-value хранилище для бота (создаётся автоматически).
Текущие ключи:
- `news_last_run_msk` — дата (YYYY-MM-DD по МСК) последнего успешного сбора новостей.
  Используется для догона пропущенных дней при перезапуске бота после 08:00.

### Лист "Описания отелей"
`Отель | Страна | Описание | Когда обновлено` — кэш описаний с tophotels.ru.
- При первом упоминании отеля в туре (если «Особенности отеля» пуст) — бот ищет
  на TopHotels и записывает результат сюда.
- На повторных публикациях того же отеля бот берёт из этого листа, сайт не дёргает.
- **Можно править вручную** — после правки бот будет использовать твою версию (а не из TopHotels).
  Это удобно: подкорректировал описание для часто публикуемого отеля — все будущие посты с ним выходят с твоим текстом.

### Лист "Журнал ошибок"
`Время | Уровень | Источник | Сообщение | Контекст` — все WARNING/ERROR/CRITICAL пишутся
автоматически из всех модулей бота (создаётся автоматически при первой ошибке).
- Время в МСК для удобства просмотра
- Уровень: WARNING / ERROR / CRITICAL
- Источник — имя логгера (publisher.vk, ai.gigachat, news.collector и т.п.)
- Старые записи (>1000) автоматически удаляются — старший конец очищается на flush
- В Telegram-боте команда `/errors [N]` показывает последние N ошибок (по умолчанию 5)
- При CRITICAL — мгновенное push-уведомление всем админам в Telegram

## HTML и стиль постов
- Telegram parse_mode=HTML принимает только: `<b>`, `<i>`, `<u>`, `<s>`, `<a>`, `<code>`, `<pre>`, `<blockquote>`, `<tg-spoiler>`, `<span>`
- `_sanitize_html_for_telegram()` ПРИНУДИТЕЛЬНО (полный список в `ai/generator.py`):
  - заменяет `<br>` / `<p>` / `<hr>` на реальные переносы строк
  - конвертирует markdown `**жирный**` → `<b>` и `*курсив*` → `<i>`
  - вырезает markdown заголовки `###` и разделители `---`/`***`/`___`
  - вырезает мета-преамбулы и подзаголовки секций
  - режет прилагательные-воду и существительные-воду по корням
  - подменяет «из родного города» на реальный город
  - обрезает блок-описание до одного предложения
  - удаляет точки в конце строк
  - гарантирует пустую строку перед хэштегами
- Применяется к выходу GigaChat и Claude перед отправкой

## Что НЕЛЬЗЯ делать
- Упоминать в постах удобства которых нет в данных (SPA, аквапарк, анимация и т.п.) — обманывать подписчиков недопустимо. Используется приоритет источников: «Особенности отеля» в Sheets → кэш «Описания отелей» → TopHotels-скрапер. Если ничего не нашли — описание остаётся общим.
- Использовать Tourvisor `search.php` или `hotel.php` — платная подписка, аккаунт не оплачен.
- Скрейпить pegast.ru напрямую — даже Beget IP получает 403 (Cloudflare). Для описаний используется tophotels.ru.
- Скипить хуки (`--no-verify`) или делать `--force push` в main без явного согласования.
- В постах новостей упоминать названия каналов-источников. Источник виден только в превью админу.

## Деплой и обслуживание

### Деплой обновлений на сервер (вручную)
```bash
# 1. С локальной машины пушим в GitHub
git push origin main

# 2. На сервере (SSH: ssh -o ServerAliveInterval=60 root@155.212.190.164)
cd /root/family-tour-bot && git pull && cd hot-tours-bot && \
  docker rm -f bot && docker build -t bot . && \
  docker run -d --name bot --env-file .env \
    -v /root/family-tour-bot/hot-tours-bot/google_credentials.json:/app/google_credentials.json \
    --restart unless-stopped bot
```

### Проверка состояния
```bash
docker ps                       # статус контейнера
docker logs bot --tail 50       # последние 50 строк логов
docker logs -f bot              # follow-режим (Ctrl+C для выхода)
docker logs bot 2>&1 | grep -i vk    # фильтр по теме
```

В Telegram у админа: `/errors [N]` — выводит последние N ошибок без захода на сервер.
