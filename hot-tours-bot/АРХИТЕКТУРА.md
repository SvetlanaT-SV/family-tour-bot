```

                      ▼
┌─────────────────────────────────┐
│   tourvisor/client.py           │

│                                 │

               │  Tour объект

               │  text + photo_url
```

## Модули и их ответственность

### `config.py`
Единственное место где читается `.env`.
Все остальные модули берут настройки отсюда — не из `.env` напрямую.

```python
Config.TOURVISOR_LOGIN      # логин Tourvisor
Config.TELEGRAM_BOT_TOKEN   # токен Telegram-бота
Config.MAX_PRICE            # максимальная цена тура
Config.PUBLISH_HOURS        # [9, 14, 19] — часы публикаций
```

---

### `tourvisor/client.py`

**Класс:** `TourvisorClient`  
**Протокол:** HTTPS GET → JSON  
**Асинхронность:** двухшаговая (запрос → polling результатов)

| Метод | Что делает |
|-------|-----------|
| `get_departure_cities()` | Список городов вылета с ID |
| `get_countries()` | Список стран с ID |
| `find_city_id(name)` | Поиск ID города по названию |
| `find_hot_tours(...)` | Главный метод — ищет и возвращает туры |

**Структура данных `Tour`:**
```
city_from, country, resort
hotel_name, hotel_stars
meal, nights, date_from
price, price_per_person
photo_url, operator, tour_id
distance_sea
```

**Базовый URL:** `https://tourvisor.ru/xml/`  
**Аутентификация:** `authlogin` + `authpass` в каждом запросе  
**Формат ответа:** JSON (`format=json`)

---

### `ai/generator.py`

**Модель:** `claude-haiku-4-5-20251001` (самая быстрая и дешёвая)  
**Стоимость:** ~0.1₽ за один пост  
**Входные данные:** объект `Tour`  
**Выходные данные:** строка — готовый текст поста

Два режима:
- `generate_post(tour, api_key)` — с Claude API (живой текст)
- `generate_post_without_ai(tour)` — шаблонный (запасной вариант)

**Шаблон поста:**
```
🔥 ГОРЯЩИЙ ТУР! {country}
✈️ Вылет: {date_from} из {city_from} ({nights} ночей)
🏨 {hotel_name} {stars}
🍽 Питание: {meal}
🌊 До моря: {distance_sea} м
💰 Цена: {price_per_person}/чел
{ai_description}     ← Claude генерирует
{ai_advantages}      ← Claude генерирует
📩 Хотите забронировать? Напишите нам!
#горящийтур #{country} #турагентствоУфа #FamilyTour
```

---

### `publisher/telegram.py`

**Класс:** `TelegramPublisher`  
**API:** Telegram Bot API (HTTPS POST → JSON)

| Метод | Что делает |
|-------|-----------|
| `publish(text, photo_url)` | Публикует пост в канал |
| `send_approval_request(...)` | Отправляет превью руководителю с кнопками |
| `notify_admin(text)` | Простое текстовое уведомление |
| `notify_new_lead(...)` | Уведомление о новой заявке |

**Формат поста:** HTML (`parse_mode="HTML"`)  
**Inline-кнопки:** `approve_{tour_id}` / `reject_{tour_id}`

---

### `publisher/vk.py`

**Класс:** `VKPublisher`  
**API:** VK API v5.199  
**Аутентификация:** access_token в каждом запросе

**Публикация фото — 4 шага:**
1. `photos.getWallUploadServer` → upload_url
2. Скачиваем фото по ссылке
3. POST на upload_url (multipart/form-data)
4. `photos.saveWallPhoto` → attachment-строка

| Метод | Что делает |
|-------|-----------|
| `_upload_photo(url)` | Загружает фото на сервера ВК |
| `publish(text, photo_url)` | Публикует пост на стене группы |

---

### `bot/handler.py`

**Библиотека:** `python-telegram-bot 20.7`  
**Паттерн:** ConversationHandler (конечный автомат)

**Состояния диалога:**
```
/start
  → ASK_NAME     (0) — "Как вас зовут?"
  → ASK_PHONE    (1) — "Ваш телефон?"
  → ASK_DATES    (2) — "Какие даты?"       [кнопки]
  → ASK_TOURISTS (3) — "Кто едет?"         [кнопки]
  → ASK_BUDGET   (4) — "Бюджет?"           [кнопки]
  → END — сохранение + уведомление руководителю
```

**Обработчик кнопок одобрения:**
- `CallbackQueryHandler` с паттерном `^(approve|reject)_`
- Проверяет `user.id == Config.TELEGRAM_ADMIN_ID`

---

### `sheets/client.py`

**Библиотека:** `gspread 6.0.2`  
**Аутентификация:** Service Account (JSON-ключ)

**Листы таблицы:**
- `Заявки` — новые лиды из бота
- `Клиенты` — расширенная база для догрева

**Колонки листа "Заявки":**
```
Дата | Имя | Телефон | Тур | Даты | Туристы |
Бюджет | Telegram ID | Username | Источник | Статус | Комментарий
```

**Статусы:** `Новая` → `В работе` → `Куплено` / `Отказ`

---

### `main.py`

**Планировщик:** `APScheduler 3.10.4` (AsyncIOScheduler)  
**Расписание:** CronTrigger — `{hour}:30` для каждого часа из `PUBLISH_HOURS`

**Два параллельных процесса:**
1. Планировщик — запускает `publish_hot_tour()` по расписанию
2. Telegram polling — бот слушает входящие сообщения

**Флаг режима:** `APPROVAL_MODE = True` → одобрение / `False` → автопилот
